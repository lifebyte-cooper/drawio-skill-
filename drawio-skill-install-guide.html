Draw.io Skill 安装通关图解

一份给内部分享使用的安装速查：先装底层引擎，再把 Skill 放进代理沙盒，最后用版本号、插件列表和别名检查把链路跑通。

1. 底层引擎能跑
2. 选择安装入口
3. 挂载进安全沙盒
4. 终端识别打通
底层引擎 三种路径 沙盒验证 故障排查

步骤 1

先把底层引擎装稳

先让终端能调用 drawio。能输出版本号，后续 Skill 才有运行地基。

  • 安装：macOS 用 Homebrew；Windows 官网安装；Linux 用 .deb/.rpm。
  • 无头 Linux 环境：额外安装 xvfb。
macOS 
brew install --cask drawio
通关验证 
drawio --version

步骤 2

三条入口，按场景选择

优先选 Claude Code 内部挂载；多工具共享选全局；熟悉目录结构再手动克隆。

路径 A 推荐 · 最简便 
/plugin marketplace add Agents365-ai/365-skills
/plugin install drawio
路径 B 全局 · 多工具共享 
npx skills add Agents365-ai/365-skills -g
路径 C 高阶 · 手动克隆 
git clone https://github.com/Agents365-ai/365-skills.git ~/.claude/skills/drawio-skill

步骤 3

下载到本地，还要挂进沙盒

npx/Git 负责下载代码；/plugin install 负责挂载到 AI 安全沙盒并授权。

  • 验证看 /plugin list，列表出现 drawio 即完成。
  • 日常维护只跑 /plugin update drawio。
大功告成验证 
/plugin list
日常更新 
/plugin update drawio

步骤 4

终端找不到，就给它开一扇别名门

终端找不到 drawio 时，用 alias 指到 macOS .app 内部的真实执行文件。

1. 打开配置文件 
nano ~/.zshrc
2. 在末尾写入映射 
alias drawio='/Applications/draw.io.app/Contents/MacOS/draw.io'
3. 刷新使其生效 
source ~/.zshrc
输入 type drawio
看到 drawio is aliased to...

检查清单通过后，说明终端已经能找到 draw.io 的真实入口。
